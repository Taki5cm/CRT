# Next Development Plan

## CRT 0.11: Training Data Expansion

목표는 학습 가능한 사건 기록을 제대로 만드는 것입니다. 단순히 급등 티커 목록을 늘리는 단계가 아니라, 각 사건의 전후 가격 경로와 결과를 같은 구조로 저장합니다.

### Completed

- `supporter-training.sqlite`에 사건 상세 구조 추가: 세션, 기준가, 고가, 거래대금, 고점 도달 시간, 후속 성과
- `+300%` 급등 뒤 `1`, `5`, `15`, `60`분 및 장 마감 결과 라벨 저장
- 급등 후 큰 하락 또는 되돌림을 초기 위험 라벨로 분류
- `+300%` 미달 후보를 대조 표본으로 저장
- 데이터셋 화면에 위험 라벨 수와 저장소 버전 표시
- 후보 검증 기록의 CSV 내보내기 제공

### Moved To 0.12

- 후보 검증 시 당시 뉴스와 SEC 공시를 같은 학습 행에 결합
- 비슷한 가격대·거래량 조건인데 급등하지 않은 종목을 대량 대조군으로 생성
- 데이터 품질 화면에 `상승 사건`, `위험 사건`, `대조군`, `자료 누락`을 더 명확히 분리

### Completion Standard

- 사용자가 등록한 과거 사건 하나가 가격 경로와 사후 결과까지 한 기록으로 저장됨
- 성공 사례만 모으지 않고 위험 라벨과 대조 표본이 함께 생성됨
- 앱 종료 후 다시 열어도 데이터셋 기록이 유지됨

## CRT 0.12: Evidence Joining, Validation And First Supporter Model

목표는 가격 경로 데이터에 당시 뉴스·SEC 근거를 결합하고, 예측값을 보여주기 전에 모델이 실제로 의미 있는지 평가하는 것입니다.

### Work Items

- 후보 검증 시 당시 뉴스와 SEC 공시를 같은 사건 행에 연결
- 희석·오퍼링 관련 SEC 양식과 이후 공시를 위험 라벨 근거로 보존
- 날짜 순서대로 학습 구간과 검증 구간을 분리해 미래 정보 누출 방지
- 입력 특징 구성: 변동률, 거래대금 가속, 가격대, 세션, 시가총액, 뉴스 유무, SEC 위험 양식
- 결과 라벨 구성: 추가 상승, 큰 되돌림, 급락 위험
- 단순 기준 모델과 ML 모델 성능 비교
- 정확도뿐 아니라 오경보율, 위험 사건 놓침 비율, 표본 부족 경고 표시
- 성능이 충분하지 않으면 앱에서 점수를 표시하지 않는 안전 장치 유지

### Completion Standard

- 시간 순서 검증 보고서가 앱 또는 내보내기 파일로 확인됨
- 검증 전 표본과 검증 후 표본이 섞이지 않음
- 사용자가 보는 점수는 성능 기준을 통과한 경우에만 제한적으로 표시됨

## Data Scale Roadmap

`100만 개 이상` 목표는 검증된 학습 행 기준으로 관리합니다.

| 단계 | 목표 |
| --- | --- |
| 초기 | 사용자가 찾은 대표 사례와 대조군으로 저장 구조 검증 |
| 확장 | 라이선스가 확인된 전체시장 역사 자료에서 극단 급등 사건 일괄 탐색 |
| 라벨링 | 가격 경로, 뉴스, SEC, 희석 가능성, 이후 결과를 결합 |
| 대규모 학습 | 사건 창과 대조 창으로 `100만+` 학습 행 생성 |
| 제품 표시 | 시간 순서 성능 검증과 오경보 검토를 통과한 결과만 표시 |

## Product Principle

CRT는 급등을 예언하거나 매수 버튼을 대신 누르는 도구가 아니라, 빠르게 움직이는 종목을 놓치지 않고 근거와 위험을 더 빨리 확인하게 돕는 리서치 서포터로 발전시킵니다. 충분한 검증을 거치기 전에는 확률이나 추천처럼 오해될 수 있는 표현을 제품에 넣지 않습니다.
