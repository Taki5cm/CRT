# Supporter ML Dataset Plan

## CRT 0.11 Objective

`CRT 0.11`의 목표는 즉시 매매 확률을 보여주는 모델이 아니라, 나중에 검증 가능한 서포터 ML을 만들 수 있도록 역사적 급등락 사건의 가격 경로와 사후 결과를 구조화해 저장하는 것입니다.

첫 수집 범위는 미국 상장 종목 중 하루 또는 장중 기준으로 `+300%` 이상 움직인 극단 급등 사건입니다. 단, 상승 성공 사례만 학습하면 실제 위험을 과소평가하게 되므로 급등 뒤 급락, 희석 공시, 유사 조건이었지만 상승하지 않은 사례도 함께 저장합니다.

## Seed Candidate Queue

사용자가 발견한 사례는 먼저 검증 대기열에 등록하고, 가격 경로가 기준을 만족한 뒤에만 학습 사건으로 확정합니다.

| 후보 | 티커 | 최초 처리 |
| --- | --- | --- |
| Allbirds / NewBird AI 관련 후보 | `BIRD` | 역사 가격과 당시 뉴스·공시 시점 검증 |
| Astrotech 관련 후보 | `ASTC` | 해당 거래일 가격 경로와 촉매 자료 검증 |
| Xiao-I 관련 후보 | `AIXI` | 티커와 `+300%` 사건 충족 여부 검증 |

기사 제목이나 사용자의 관찰만으로는 사건을 확정하지 않습니다. 데이터 제공자의 체결·분봉 자료와 당시 이용 가능했던 뉴스·공시만으로 포함 여부를 결정합니다.

## Event Definition

`extreme_daily_spike`는 전 정규장 종가 대비 해당 거래일 장중 고가 또는 종가가 `+300%` 이상인 사건입니다. 이 표본은 역사적 극단 사례 연구에 사용합니다.

`realtime_trigger_event`는 사용자가 설정한 초·분 단위 변동률, 최소 가격, 거래대금 조건을 통과한 실제 CRT 포착입니다. 이 표본은 알림 품질과 사후 결과를 평가하는 데 사용합니다.

프리마켓, 정규장, 애프터마켓은 서로 다른 세션으로 저장합니다. 세션을 섞으면 동일한 움직임이라도 유동성과 체결 가능성이 다른 문제를 숨기게 됩니다.

## Training Corpus

`100만 개 이상의 급등주 데이터` 목표는 `300% 상승 종목 100만 개`가 아니라, 검증된 사건과 비교 표본에서 만들어지는 `100만 개 이상의 라벨 행`으로 정의합니다.

| 표본 종류 | 목적 |
| --- | --- |
| 극단 상승 표본 | `+300%` 이상 사건의 전조와 지속 패턴 학습 |
| 위험 표본 | 포착 뒤 급락, 큰 되돌림, 희석·오퍼링 관련 공시가 확인된 사건 학습 |
| 대조 표본 | 가격대·시가총액·유동성·뉴스 조건은 비슷했지만 급등하지 않은 사례 학습 |
| 일반 구간 표본 | 모델이 모든 작은 움직임을 급등으로 오인하지 않도록 기준선 제공 |

## Stored Fields

감지 시점 이전 또는 당시 이미 이용 가능했던 정보:

- 티커, `CIK`, 거래일, 세션, 감지 시각
- 기준 가격, 감지 가격, 변동률, 거래대금, 거래량 가속도
- 시가총액, 발행주식 수, 허가된 경우 검증된 유통주식 수
- 감지 당시 공개되어 있던 뉴스와 SEC 공시 양식

사후 결과 라벨:

- 감지 후 `1`, `5`, `15`, `60`분 및 장 마감 수익률
- 최대 추가 상승폭, 최대 하락폭, 고점 도달까지 걸린 시간
- 다음 거래일 갭과 종가 결과
- 사건 이후 확인된 희석·오퍼링 관련 공시 여부

감지 뒤 발표된 뉴스나 공시는 예측 입력에 넣지 않고 결과 분석에만 사용합니다. 그래야 모델이 미래 정보를 미리 아는 것처럼 보이는 오류를 막을 수 있습니다.

## Data Sources And Scale

| 목적 | 후보 소스 | 원칙 |
| --- | --- | --- |
| 전체시장 역사 사건 탐색 | Massive Stocks Flat Files 또는 동등한 라이선스 데이터 | 전체 거래소를 포함하는 역사 자료로 후보를 찾음 |
| 분봉·체결 경로 검증 | Massive 또는 Alpaca Market Data | 조정 여부와 세션을 함께 기록 |
| 뉴스 근거 | Alpaca News 및 사람이 확인할 외부 링크 | 감지 시점 이전 공개 여부를 저장 |
| 공시·희석 라벨 | SEC EDGAR submissions 및 원문 공시 | 공식 제출 시각을 기준으로 사용 |

무료 Alpaca `IEX` 피드는 초기 시험에는 유용하지만 한 거래소 자료만 포함하므로, 전체 미국 시장의 역사적 학습 표본과 초단위 상용 감지를 대표할 수 없습니다. 전체시장 학습 또는 유료 서비스로 이어질 때는 `SIP` 수준 데이터나 전체시장 역사 파일의 저장·재배포·파생 결과 표시 권한을 계약상 확인해야 합니다.

SEC는 인증 키 없이 회사별 제출 내역을 제공하며, 대규모 처리는 공개 제출 이력 ZIP 자료로 구축할 수 있습니다. 단, 자동 요청은 SEC 접근 정책과 요청 식별 규칙을 따라야 합니다.

## Implemented In CRT 0.10

1. 실시간 포착 기록과 분리된 `supporter-training.sqlite` 로컬 학습 후보 저장소를 추가했습니다.
2. 사용자가 티커와 거래일 후보를 등록해 검증 대기열에 넣을 수 있습니다.
3. Alpaca IEX 역사 분봉으로 직전 수신 거래일 종가 대비 장중 고가의 `+300%` 조건을 계산하고, 사건 또는 대조 표본 상태로 저장합니다.
4. 화면에는 후보 수, 확인된 300% 사건 수, 대조 표본 수를 표시합니다.
5. 충분한 검증 전에는 상승 기대률이나 매매 추천을 표시하지 않습니다.

## Implemented In CRT 0.11

1. `supporter-training.sqlite`를 스키마 버전 `2`로 마이그레이션하고 기존 후보를 보존합니다.
2. 후보 검증 결과에 거래일 시가·종가·총 거래량·추정 거래대금·고점 시각·세션·고점 도달 시간을 저장합니다.
3. 고점 이후 `1`, `5`, `15`, `60`분 성과와 장 마감 기준 고점 대비 되돌림을 저장합니다.
4. 급등 후 흐름을 `강세 유지형`, `되돌림 경계`, `고위험 급락형` 등 학습용 위험 라벨로 분류합니다.
5. 후보와 검증 결과를 CSV로 내보내 외부 분석 또는 백업에 사용할 수 있습니다.

## Next Dataset Expansion

1. 라이선스가 확인된 전체시장 역사 데이터로 IEX 검증 표본을 재검사합니다.
2. 당시 뉴스와 당시 SEC 공시를 가격 경로 기록에 결합합니다.
3. 유사 조건 실패 대조군을 대량 생성합니다.
4. 누락된 데이터 소스와 표본 편향을 확인하는 데이터 품질 보고를 확장합니다.
